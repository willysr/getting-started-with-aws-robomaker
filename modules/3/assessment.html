    Assessment for Module 3 - AWS RoboMaker Development Environment

    AWS RoboMaker setup

    This assignment is to help you self-assess your understanding on AWS RoboMaker setup and run. Read carefully each question. Once answered, you will receive feedback. Good luck!

    Q1

    To set up AWS RoboMaker, the required steps are: (select all that apply)

    * Set up S3 bucket.
    * Set up the IAM permissions.
    * Set Cloud9 for RoboMaker.
    * Set Kinesis.
    * Set Polly.

    Q2

    The workflow to develop and deploy software that has been written is:

    1. 1. Build, 2. Bundle, 3. Run.
    2. 1. Bundle, 2. Build, 3. Run.
    3. 1. Run, 2. Build, 3. Bundle.
    4. 1. Build, 2. Run, 3. Bundle.
    5. 1. Run, 2. Bundle, 3. Build.

    Q3

    Running the simulation indefinitely does not affect the cost.

    1. True.
    2. False.

    Q4

    A ROS package to be built, bundled, and run within AWS RoboMaker, should be in:

    1. the src folder of a colcon workspace.
    2. just having an src folder in the home directory.
    3. It doesn't matter where the ROS package is.
    4. GitHub.

    Q5

    You are welcome now to explore some of the existing packages, e.g., from GitHub, and try to run them on AWS RoboMaker. We will discover what is under the hood of ROS so that we can start developing exciting behaviors for the robot.
